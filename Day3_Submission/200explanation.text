When I tried to merge the feature/divide-function branch into main after already merging feature/multiply-function, Git found a merge conflict in src/main.py. The conflict happened in two places: the version comment at the top of the file and the print statement inside the main block.
This happened because both branches changed the same lines, but in different ways. The multiply branch updated the version to 1.1.0 and added a multiplication test in the __main__ section. The divide branch, which was created from the original main, updated the version to 1.0.1 and added a division test. Since both branches edited the same parts of the file, Git could not automatically choose which changes to keep.
To fix this, I opened the file and looked at the conflict markers (<<<<<<< HEAD, =======, >>>>>>>). Instead of choosing one side and deleting the other, I combined the changes. I updated the version to 1.2.0 to show that both features are now included. I also kept both the multiply and divide test lines in the main block so that all four calculator functions run correctly.
This way, I kept all the work from both branches and made sure the final version was clean, complete, and easy to understand.
